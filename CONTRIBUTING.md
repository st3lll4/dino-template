# Contributing

Bug fixes and new features are both welcome.

## Branches

Use Conventional Commits format:

```
feat(scope)/short-description
fix(scope)/short-description
chore(scope)/short-description
```

Examples: `feat(messaging)/add-broadcast`, `fix(hmr)/reconnect-loop`

## Code style

TypeScript and ESLint are configured. Run before submitting:

```bash
npm run lint
npm run typecheck
```

## Pull requests

No issue needed — open a PR directly. Use the PR template and keep the description clear.

PR titles should follow the same Conventional Commits format: `feat(scope): short description`.
